## Athena
![](https://velog.velcdn.com/images/sorkdlrlsek/post/77a49fc2-539d-4426-a014-6901053ac60d/image.png)

- DataBase 생성
```
CREATE DATABASE IF NOT EXISTS sales_db;
```

- Table 생성
```
CREATE EXTERNAL TABLE IF NOT EXISTS sales_db_table (
	order_id  STRING,
    order_date STRING,
    region	 STRING,
    product	STRING,
    amount	STRING
)
LOCATION '{배포 파일이 있는 s3 폴더의 경로}'
```


- 쿼리
```
SELECT region, SUM(amount) AS total_sales
FROM sales_db.sales
WHERE CAST(order_date AS DATE) >= DATE '2024-01-01'
  AND amount >= 100000
GROUP BY region
ORDER BY total_sales DESC;
```
```
SELECT product, SUM(amount) AS total_sales
FROM sales_db.sales
WHERE CAST(order_date AS DATE) >= DATE '2024-01-01'
GROUP BY product
ORDER BY total_sales DESC
LIMIT 5;
```



| 구문 | 설명 | 응용 |
|---|---|---|
| SELECT region, SUM(amount) | 지역명과 금액 합계 조회 | SELECT product, COUNT(*) → 제품별 주문 수 |
| AS total | 컬럼 이름을 total로 표시 | AS 매출합계, AS cnt |
| FROM sales | sales 테이블에서 가져옴 | FROM orders, FROM users |
| WHERE order_date >= '2024-01-01' | 2024년 1월 1일 이후만 | >= '2023-06-01' → 2023년 6월 이후 |
| AND amount >= 100000 | 10만원 이상인 것만 | AND amount >= 50000 → 5만원 이상 |
| GROUP BY region | 지역별로 묶어서 집계 | GROUP BY product → 제품별 집계 |
| ORDER BY total DESC | 합계 높은 순 정렬 | ORDER BY total ASC → 낮은 순 |

---

| 구문 | 설명 | 응용 |
|---|---|---|
| SUM(amount) AS total_sales | 제품별 매출 합계 | AVG(amount) → 평균 금액 |
| GROUP BY product | 제품별로 묶어서 집계 | GROUP BY region → 지역별 |
| ORDER BY total_sales DESC | 매출 높은 순 정렬 | ORDER BY total_sales ASC → 낮은 순 |
| LIMIT 5 | 상위 5개만 표시 | LIMIT 10 → 상위 10개 |

---

| 구문 | 설명 |
|---|---|
| ORDER BY total DESC | 매출 높은 순 (1등부터) |
| ORDER BY total ASC | 매출 낮은 순 (꼴찌부터) |
| ORDER BY order_date DESC | 최신 날짜 순 |
| ORDER BY order_date ASC | 오래된 날짜 순 |
| ORDER BY amount DESC | 금액 높은 순 |
| ORDER BY region ASC | 지역명 가나다 순 |
| ORDER BY product DESC | 제품명 역순 |

---

| 함수 | 설명 | 예시 |
|---|---|---|
| SUM(amount) | 합계 | 총 매출 |
| AVG(amount) | 평균 | 평균 주문 금액 |
| COUNT(*) | 행 개수 | 총 주문 건수 |
| MAX(amount) | 최댓값 | 최고 주문 금액 |
| MIN(amount) | 최솟값 | 최저 주문 금액 |
